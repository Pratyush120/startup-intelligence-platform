# ADR 005: FastAPI Decision

## Status
Accepted

## Problem
We need to expose the intelligence data generated by the SDIP pipeline to a frontend dashboard. The API needs to be fast, self-documenting, and type-safe to ensure smooth integration with the Next.js React-Query architecture.

## Decision
We chose **FastAPI** as the web framework.

## Alternatives Considered
- **Flask**: Easy to use but lacks built-in async support and automatic OpenAPI documentation. Pydantic integration is manual.
- **Django / Django Rest Framework (DRF)**: Extremely heavy and opinionated. Since we are using a custom Repository pattern and SQLite, Django's ORM and admin panel would add unnecessary friction and override our architecture.
- **Express.js (Node)**: Requires rewriting the data layer in TypeScript. Keeping the backend in Python allows seamless integration with the data science and LLM orchestration logic.

## Consequences
- **Positive**: Automatic, interactive OpenAPI documentation (Swagger UI), massively speeding up frontend development.
- **Positive**: Pydantic models validate all incoming and outgoing data, providing strict type safety.
- **Positive**: Built-in Dependency Injection (`Depends()`) perfectly supports injecting our Repository and configuration classes cleanly.
