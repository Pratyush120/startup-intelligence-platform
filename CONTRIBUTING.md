# Contributing to SDIP

First off, thank you for considering contributing to the Startup Data Intelligence Platform! It's people like you that make SDIP such a powerful tool.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.
- Use the **Bug Report** issue template.
- Explain the problem, include the steps to reproduce, and mention what you expected to happen.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.
- Use the **Feature Request** issue template.
- Provide a clear and descriptive title for the issue.
- Explain why this enhancement would be useful.

### Pull Requests
The process described here has several goals:
- Maintain SDIP's quality
- Fix problems that are important to users
- Engage the community in working toward the best possible SDIP

1. **Fork the repo** and create your branch from `main`.
2. **If you've added code that should be tested, add tests.**
3. **Ensure the test suite passes.**
4. **Make sure your code lints.**
5. **Issue that pull request!**

## Development Setup
Please see the `README.md` for our Quickstart guide. Ensure you have the proper pre-commit hooks installed if you are making changes to the backend or frontend code.

## Styleguides

### Git Commit Messages
- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests liberally after the first line
